## Session: Go Structs & Real-World Connection

**What we did:**
- Started Go structs lesson with EC2 instance example
- Malik typed out a struct definition (Instance with ID, CPU, Mem, State fields)
- Created an instance, printed its ID — code ran successfully
- Explained what an EC2 instance actually is ("virtual computer in a data center with CPU/RAM/ID/state")
- **BREAKTHROUGH:** Malik made the connection: "If I define a struct with specs and Terraform applies it, it creates a real server for me through AWS instead of using the console"
- He understood the value prop of IaC: repeatable, trackable, destroyable (prevents surprise AWS bills), version-controlled

**Key learning moment:**
Malik had been confused by abstract IaC benefits, but concrete explanation of the struct→Terraform→AWS flow made it click. He now sees why defining infrastructure as code is better than clicking around the console.

**What's next:**
- Practice creating multiple structs/instances (currently working on second instance)
- Methods on structs (functions that operate on struct data)
- Then back to Terraform with this foundation
